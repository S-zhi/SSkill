# 认知报告规格（四部分）

报告目标：把一段口播 ASR 提炼成「能提升认知」的速读材料，成品控制在 **1~2 页 A4**，供每日阅读。
全文用中文。四个部分依次排布，标题固定用 `##`。

下面是每部分的写法规格，末尾附一个完整示例。生成 Markdown 时严格按此结构。

---

## 说话人拆分（前置步骤，不进正文）

在动笔写报告前，先判断口播形态并拆分文本。这一步的产物是「整洁的分句/分轮文本」，用于你自己理解，不直接作为报告的一节。

判断依据：通读 ASR，看是否存在明显的问答、称呼、身份切换、"你说/我觉得/主持人问"等信号。
- **单播（最常见）**：一个人从头讲到尾。按句号/语义把长段落拆成一句一句，去掉口水词（"就是""那个""然后呢"等冗余）。
- **双播**：两个人对话（如访谈、对谈）。按说话人分轮，标注 `A:` / `B:`（能判断身份就用身份，如 `主持人:` / `嘉宾:`）。
- **多播（极少）**：三人及以上，按 `A/B/C` 分轮。

拆分只为帮助你理解与后续摘录，**不必把全部分句塞进报告**。

---

## 第一部分：核心提炼

标题：`## 一、核心提炼`

包含三样东西：

1. **中心思想**：一句话点破整段口播真正想传达的核心意图（不是复述话题，是提炼观点）。用引用块 `>` 呈现，使其醒目。
2. **要点**：根据文本强度拆成 **1~3 点**，最多 3 点。强度弱就合并成 1 点，别硬凑。每点一句话，可加粗关键词。
3. **分区 + 标签**：把内容归到 **约 5 个、不超过 5 个、也不要过少（建议 3~5）** 的类别里，给每个分区打一个标签。
   - 标签用 `<span class="tag">标签名</span>` 包裹，写在分区名旁。
   - 分类方向示例（按内容自由拆分，不限于此）：为人处事、Coding 经验/范式、经济头脑、认知方法、情绪管理、决策模型……
   - 每个分区下用一句话说明该区在讲什么。

推荐用一个小表格承载「分区 + 标签 + 一句话含义」，紧凑清晰。

---

## 第二部分：精华摘录

标题：`## 二、精华原文`

从第一部分理解的基础上，回到（已拆分优化的）口播文本，**截取有价值的原文段落**：
- 讲得好的**故事、案例、有逻辑链条的推理**，保留下来，不要删光。
- 表达本身有魅力的整段，可**原封不动整段收录**。
- 摘录经过第一步的分句/去口水词优化即可，允许轻微顺句，但**保留原意与原味**，不要改写成白话总结。

每段摘录用 `<div class="excerpt">…</div>` 包裹。控制在 1~3 段，服从整体 1~2 页的篇幅。

---

## 第三部分：印证与批判

标题：`## 三、印证与批判`

1. **一句话总结** 本部分（对整篇思想的凝练判断）。
2. **引用两句名人名言**，借名言映射出的哲理来印证该思想的可行性——**避免纯白话**，让印证有分量。名言用引用块 `>` 呈现，注明出处/作者。
   - 名言要真实、准确、与思想贴合；宁可用广为人知的经典，也不要杜撰。
3. **先正向证明**：说明这套思想为何成立/可用。
4. **二次批判（若有）**：若发现文章有纰漏、以偏概全、或总结不到位之处，写出批判，放在本部分。用 `<div class="critique">…</div>` 包裹。没有明显问题可省略批判，但要诚实，不要强行挑刺也不要一味吹捧。
5. **评级方块**：文末放一个可勾选的小方块（打印后手动打勾），HTML 固定如下：

```html
<div class="rating">
  <span class="rating-title">我的评级：</span>
  <span class="box"><span class="checkbox"></span>认同</span>
  <span class="box"><span class="checkbox"></span>不认同</span>
</div>
```

---

## 第四部分：延伸应用

标题：`## 四、延伸应用`

把思想落到可执行的场景。分 1~3 个方向，**每个方向举 2~3 个 Case**。
Case 要具体、可操作，最好贴合读者的实际（如工程实践、职业发展、日常决策），而不是空泛口号。
用 `###` 小标题分方向，Case 用短列表。

---

## 篇幅与排版纪律

- 成品 **1~2 页**。若内容超长，优先压缩第二部分摘录数量、第四部分 Case 数量，保住第一、三部分。
- 少用装饰，多用信息密度。标签、表格、引用块、excerpt/critique/rating 这些结构化元素是主要视觉锚点。
- 顶部第一行是 `# 标题`，其下可用 `<p class="meta">来源 / 日期</p>` 放元信息。

---

## 完整示例（结构参考，非真实内容）

```markdown
# 认知报告 · 长期主义为什么难

<p class="meta">来源：抖音口播 · 单播 · 2026-07-16</p>

> 中心思想：长期主义不是"看得远"，而是有能力承受短期不被理解的代价。

## 一、核心提炼

**要点**
- 长期主义的稀缺，本质是**延迟满足能力**的稀缺。
- 复利只对"不中断"的人生效，中断一次就前功尽弃。

**分区一览**

| 分区 | 标签 | 一句话 |
|---|---|---|
| 为人处事 | <span class="tag">为人处事</span> | 忍住解释的冲动，用结果说话 |
| 经济头脑 | <span class="tag">经济头脑</span> | 复利思维要求不中断的持续投入 |
| 认知方法 | <span class="tag">认知方法</span> | 用长周期视角重估当下得失 |

## 二、精华原文

<div class="excerpt">
他说他刚开始做的时候，三年没人看得懂，家里人劝他别折腾了。他没解释，就一直做。第四年别人回过头才发现，原来他早就在等这个拐点。
</div>

## 三、印证与批判

一句话总结：真正的长期主义，是把"熬"变成一种可复利的能力。

> "复利是世界第八大奇迹。" —— 常被归于爱因斯坦

> "行胜于言。" —— 清华校训

正向来看，复利模型在时间足够长时会拉开指数级差距，这为长期投入提供了数学依据。

<div class="critique">
批判：口播回避了"方向错误的长期主义等于加速失败"这一前提。长期主义的收益依赖方向正确，视频未做区分，易误导。
</div>

<div class="rating">
  <span class="rating-title">我的评级：</span>
  <span class="box"><span class="checkbox"></span>认同</span>
  <span class="box"><span class="checkbox"></span>不认同</span>
</div>

## 四、延伸应用

### 工程实践
- 对一个开源项目做每日小步提交，用连续性换复利，而非一次性冲刺。
- 重构时优先偿还高频路径的技术债，让收益持续累积。

### 职业发展
- 用 3 年视角选赛道，而非用 3 个月薪资差跳槽。
- 建立"不被理解也继续"的证据日志，减少中途放弃。
```
