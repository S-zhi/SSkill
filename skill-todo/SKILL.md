# 目标

本 Skill 用于长期收集 Skill 的改进意见。

当用户表达某个 Skill 存在不足、需要优化或未来想修改时，不要立即修改 Skill，而是将意见追加记录到对应的 Markdown 文件中。

本 Skill **只负责记录改进意见**，**绝不负责修改 Skill**。

---

# 存储位置

所有记录统一存放到：

```text
~/.skill-todo/
```

每个 Skill 使用一个独立的 Markdown 文件。

例如：

```text
~/.skill-todo/
├── README-Generator.md
├── translator.md
├── skill-todo.md
├── remote-awesome-skill.md
└── ...
```

文件名应尽可能与 Skill 名称保持一致。

如果文件不存在，则自动创建。

如果文件已存在，则在文件末尾追加。

**绝不能覆盖已有内容。**

---

# 每条记录必须包含的信息

## 1. 问题名称

使用一句话概括需要改进的问题。

要求：

- 简洁
- 明确
- 能作为标题阅读

例如：

- Quick Start 放置位置不合理
- Prompt 过长
- 示例不足
- Windows 支持说明缺失

---

## 2. 解决方案

本部分始终包含两个部分。

### 用户建议

记录用户提出的解决方案。

如果用户没有提供解决方案，则填写：

> （用户未提供）

不得根据上下文猜测用户想法。

### AI 建议

AI 必须针对该问题给出一个合理、可执行的改进建议。

要求：

- **无论用户是否提供了解决方案，都必须生成 AI 建议。**
- AI 建议可以与用户建议一致，也可以提供另一种思路。
- 如果与用户建议不同，不要评价用户方案是否正确，仅提供另一种可参考的方案。
- AI 建议应尽量具体，可直接作为未来修改 Skill 的参考。

---

## 3. 实际例子

优先从当前聊天内容中提取真实案例。

例如：

用户正在讨论 README Pipeline，则直接提取当前聊天中的内容作为例子。

如果当前聊天没有可引用的案例，则说明：

该问题为什么会影响 Skill 的使用体验。

不要虚构不存在的案例。

---

# 写入格式

始终按照以下格式追加：

```markdown
## Issue #<自动编号>

时间：<当前时间>

### 问题名称

...

### 用户建议

...

### AI 建议

...

### 实际例子

...

---
```

---

# 工作规则

必须遵循以下原则：

1. 永远追加。

绝不能覆盖已有内容。

2. 永远不要删除历史记录。

3. 永远不要修改历史记录。

4. 每一个独立的问题都应生成一个新的 Issue。

不要将多个无关的问题合并。

5. 尽量保留用户原话。

AI 仅负责整理表达、补充细节和生成 AI 建议，不得改变用户原意。

6. 如果能够从当前聊天提取真实案例，则优先提取。

不要为了补充内容而编造案例。

7. 如果用户没有明确说明 Skill 名称，则根据当前聊天上下文进行推断。

只有完全无法推断时，才询问用户。

8. 本 Skill 不负责修改任何 Skill。

本 Skill 的职责仅为：

- 收集问题
- 保存问题
- 为未来统一修改提供参考

9. 每次记录都必须保持独立。

即使多个 Issue 内容相似，也不要自动合并。

10. 不要因为问题已经存在，就忽略新的记录。

不同时间、不同场景产生的反馈，都具有保留价值。

# 示例

用户：

README 的 Quick Start 放得太后面了，希望放到前面一点。

追加记录：

```markdown
## Issue #7

时间：2026-07-01 20:31

### 问题名称

Quick Start 放置位置不合理。

### 用户建议

将 Quick Start 调整到 README 前半部分。

### AI 建议

建议调整 README 结构为：

项目简介 → Quick Start → Features → Installation → Usage。

同时增加一个一分钟快速开始示例，降低首次阅读成本。

### 实际例子

当前 README 顺序为：

Logo → Badge → Features → Installation → Quick Start

建议修改为：

Logo → Badge → 项目简介 → Quick Start → Features → Installation

---
```

# 禁止行为

禁止覆盖已有文件。

禁止删除历史记录。

禁止修改已有 Issue。

禁止整理历史 Issue。

禁止自动合并多个历史 Issue。

禁止直接修改目标 Skill。

禁止因为问题重复而忽略新的记录。

本 Skill 的唯一职责，是持续积累 Skill 的改进意见，为未来统一优化提供依据。
